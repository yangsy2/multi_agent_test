
입력 내용

Ollama 와 Langchain, Lang graph 를 이용해서 다중 Agent 를 만들려고 해. 
먼저 Test 용 다중 Agent 를 만들거야. 
그리고 여기에서 사용할 MCP tool 들도 임시로 만들어야 해. 
만들Agent  와 tool 에 대한 설명은 아래에서 별도로 할거야. 
MCP tool 들은 주어진 리스트에서 random 으로 값을 선택하여 하위 단계의 Agent 에게 돌려주고, 
하위 Agent  들은 여기에서 얻어진 값을 기준으로 판단하여 중간 Agent 에게 돌려주고, 
중간 Agent  들은 여기에서 얻어진 값을 기준으로 판단하여 상위 Agent 에게 돌려주고, 
상위 Agent 는 이 답들을 모아서 정리해주어야 해.

상위 Agent  
. 이름 :  이상분석 
. 역할 : DEFECT 분석이 들어오면 DEFECT 명 기준으로 관련된 공정을 찾아 이상 이유를 찾아 확인하고 
이상 LOT 으로 분석이 들어오면 하위 공정 Agent 의 답변들을 바탕으로 유의한 원인을 찾아주는 업무
. DEFECT 명 기준 관련 공정
DEFECT	관련 공정
PTN_ERR	PHT, DRY
BLACK_SPOT	CVD, SPT, WET
CORROSION	DRY, WET
PTN_OPEN	PHT
SCRATCH	WET, SPT
HABU_YUHAK	PHT, SPT
FRECKLE	ELA, WET, PHT
기타	PHT, DRY, CVD, SPT, WET (전체)



중간 Agent  
. 이름 : PHT, DRY, CVD, SPT, WET, ELA
. 역할 : 각각 한 공정을 의미하고, 하위 Agent 들에게 차례로 돌아가며 답변을 얻고, 알려준 답변을 바탕으로 해당 공정이 이상이 있는지 없는지, 무엇때문에 이상이 있는지  판단하여 상위  Agent  에게 알려주는 역할
          만일 해당공정에 이상이 없다면, 상위 Agent 에게 다른 공정을 확인해 보라고 전달해주어야 함

하위 Agent
. 이름  :  EQP, Chamber, Layer, MODEL 
. 역할  :  각각 이름에 맞는 분석 단계를 운영하며, tool 에서 보내준 DATA 를 이용하여 해단 단계가 이상이 있는지 판단하여, 이상 유무를 중간 Agent 에게 알려주는 역할
             이상이 없는 기준은 TREND는 일정, MAP 은 특이 MAP 없음, FDC는 이상없음, cTTTM 은 변경점 없음 인 경우에 한정
